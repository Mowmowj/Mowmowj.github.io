this是java 关键字 意思：这个

this是一个引用

this是一个变量

this保存了内存地址指向了自身

this 保存在JVM堆内存中

this应用在两个位置

构造方法当中

实例方法当中

（在静态方法种无法使用this（因为没有对象的参与，静态方法是采用类名的方式调用）所以在静态方法中无法直接使用this

在实例方法中可以用，因为调用实例方法的时候调用了对象。

方法重载 Overload

方法重载是指在一个类中定义多个同名的方法

什么时候使用方法重载？

覆盖要求参数列表相同

方法重载规范

一.方法名一定要相同。

二.方法的参数表必须不同，包括参数的类型或个数，以此区分不同的方法体。

1.如果参数个数不同，就不管它的参数类型了！

2.如果参数个数相同，那么参数的类型必须不同。

三.方法的返回类型、修饰符可以相同，也可不同。

方法重载和什么无关？

和返回值类型无关和修饰符列表无关

方法覆盖/方法重写（Override/Overwrite）

什么时候使用方法重写？

\1. 当父类中的方法已经无法满足子类的业务需求时

\2. 子列有必要将从父类继承过来的方法进行重新编写

\3. 这个重写的过程称为方法重写/方法覆盖

什么条件满足情况下发生重载？

\1. 方法重写发生在具有继承父子列关系之间

\2. 方法重写时，返回值类型相同、方法名相同、形参列表相同（建议复制粘贴）

\3. 方法重写时，访问权限可以更大不能更低

\4. 方法重写时，方位异常不能更多可以更少

注意

\1. 私有方法不能继承，所以不能方法重写

\2. 构造方法不能继承，所以不能方法覆盖

\3. 静态方法不存在覆盖

\4. 覆盖只针对方法，不谈属性和变量

Final关键字

Final是一个关键字 表示最终的不可变的

Final修饰的类无法被继承

Final修饰的方法无法被覆盖

Final修饰的变量一旦复制以后不可重新赋值

Final修饰的实例变量 实例变量有默认值 用final修饰必须手动赋值

Final 修饰的实例变量是不可变的这种变量一般和static联合使用被称为常量

Public static final 类型 常量名 = 值

JAVA 要求常量名全部大写 单词之间用__下划线连接

Final修饰符的引用一旦指向某个对象后不能指向其他对象，但是所指向对象的内存是可以被修改的。
